## 1. システムアーキテクチャ

### 1.1 全体構成

| コンポーネント | 技術 | 役割 |
|----------------|------|------|
| フロントエンド | React 18 + TypeScript（Vite） | UI描画・状態管理・API通信 |
| バックエンド | Spring Boot 3.x + Java 21 | ビジネスロジック・認証・データ処理 |

```
Browser (React SPA)  →  Spring Boot REST API  →  MySQL
```

> ※本来の構成：Windows(ブラウザ/Excel) → Linux(Spring Boot/Tomcat) → MySQL

### 1.3 バックエンドレイヤー構成

| レイヤー | クラス群 | 責務 |
|----------|----------|------|
| Controller | `*Controller.java` | HTTPリクエスト受付・レスポンス返却・`@PreAuthorize`（ロール制御） |
| Service | `*Service.java` | ビジネスロジック・`@Transactional` トランザクション管理 |

## 2. 認証・セキュリティ設計

### 2.1 JWT認証フロー

1. `POST /api/auth/login` に loginId / password を送信
2. Spring Security が UserDetailsService でアカウント照合
3. BCrypt でパスワード検証 → 成功時 JWT を生成して返却
4. 以降のリクエストは `Authorization: Bearer <token>` を付与
5. JwtAuthenticationFilter がトークンを検証（**1次：FilterChain**）
6. `@PreAuthorize` でロール別アクセス制御（**2次：メソッドレベル**）

### 2.2 JWT仕様

| 項目 | 値 |
|------|----|
| アルゴリズム | HS256 |
| 有効期限 | 30分 |
| ペイロード | sub（loginId）/ role / iat / exp |
| 二重検証 | FilterChain（1次）+ @PreAuthorize（2次） |
| パスワード | BCrypt ハッシュ化（平文保存禁止） |

### 2.3 ロール定義

| ロール | 権限 |
|--------|------|
| ADMIN | 全機能アクセス / 社員管理 / アカウント発行 / 権限管理 / Excel取込 / 全社員データ閲覧 |
| USER | 自身の打刻 / 勤怠確認 / 休暇申請 / 自身の集計確認 |

## 4. データベース設計

### 4.1 DB構成

- **DB名**：`kintai_db`
- **テーブル数**：8テーブル

### 4.2 テーブル一覧

| テーブル名 | 用途 |
|------------|------|
| `employee` | 従業員マスタ（氏名・部署・雇用形態・在職フラグ） |
| `account` | 認証アカウント（loginId・パスワードハッシュ・ロール） |
| `work_time` | 勤怠打刻データ（出勤・退勤・外出・戻り・業務メモ） |
| `batch_import_history` | Excel/CSVインポート履歴（ファイル名・総件数・成功/エラー件数） |
| `leave_request` | 休暇申請（区分・期間・ステータス・承認者） |
| `holiday` | 祝日マスタ（日本祝日API取得データ） |

### 4.3 主要インデックス

| テーブル | カラム | 目的 |
|----------|--------|------|
| `work_time` | `employee_id, work_date` | 複合インデックス — 月次集計・一覧表示の高速化（N+1解消と併用） |
| `account` | `login_id` | UNIQUE — ログインID重複防止 |
| `leave_request` | `employee_id, status` | 休暇一覧フィルタ高速化 |

### 4.4 N+1問題対応

JOIN FETCH を使用し、月次勤怠一覧取得時のクエリ数を最適化。

| 最適化前 | 最適化後 |
|----------|----------|
| 21クエリ（N+1問題発生） | 1クエリ（JOIN FETCHによる最適化） |

## 5. 実装上の工夫点

| 工夫点 | 詳細 |
|--------|------|
| バリデーション | CSV取込時の行単位エラー検出・成功/失敗件数を明示報告 |
| エラーハンドリング | GlobalExceptionHandler実装（BusinessException階層・スタックトレース非公開） |
| Serviceレイヤー分離（SRP） | Controller約70行・ビジネスロジックをService層に集約 |
| N+1解消 | JOIN FETCHにより21クエリ → 1クエリに最適化 |
| 再利用性・セキュリティ | AuthContext（props drilling解消）・複合インデックス・JWT二重保護（FilterChain + @PreAuthorize） |
