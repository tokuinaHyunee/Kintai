## 3. 主要クラス詳細設計

### 3.1 JwtUtil（JWT生成・検証）

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `generateToken(loginId, role)` | `String` | JWT生成（有効期限30分・HS256） |
| `validateToken(token)` | `boolean` | トークン検証（署名・有効期限チェック） |
| `getLoginId(token)` | `String` | トークンからloginId抽出 |
| `getRole(token)` | `String` | トークンからロール抽出 |

### 3.2 WorkTimeService（勤怠ビジネスロジック）

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `clockIn(employeeId)` | `WorkTimeDto` | 出勤打刻。当日レコード作成・二重打刻チェック |
| `clockOut(employeeId)` | `WorkTimeDto` | 退勤打刻。出勤未打刻の場合 BusinessException 送出 |
| `goOut / returnFromOut(employeeId)` | `WorkTimeDto` | 外出・戻り打刻 |
| `getMonthlyList(employeeId, year, month)` | `List<WorkTimeDto>` | 月次勤怠一覧取得（JOIN FETCHでN+1解消） |
| `importExcel(file, employeeId)` | `ImportResult` | Excel取込。列名マッピング→バリデーション→UPSERT |
| `getTodayStatus(employeeId)` | `ClockStatus` | 本日打刻状態返却（ボタン制御用） |
| `getMonthlySummary(employeeId, year, month)` | `WorkTimeSummary` | 月次集計（稼働率・労働時間）算出 |

### 3.3 GlobalExceptionHandler

`@RestControllerAdvice` によりアプリケーション全体の例外を一元ハンドリング。スタックトレースは外部に公開しない。

| ハンドラ | HTTPステータス | 対象 |
|----------|--------------|------|
| `handleBusinessException` | 400 | 二重打刻・ビジネスルール違反 |
| `handleValidationException` | 400 | `@Valid` 入力値バリデーションエラー |
| `handleImportException` | 400 | Excel/CSVインポートエラー（行単位エラー詳細付き） |
| `handleAuthException` | 401 | JWT無効・認証失敗 |
| `handleNotFoundException` | 404 | リソース未存在 |
| `handleGeneral` | 500 | 予期しないエラー（スタックトレース非公開） |

## 4. フロントエンド詳細設計

### 4.1 AuthContext（props drilling解消）

React Context API を使用して JWT・ロール・employeeId をグローバル管理する。

| 項目 | 内容 |
|------|------|
| 管理状態 | `token`（JWT文字列）/ `role`（ADMIN \| USER）/ `employeeId` |
| `login()` | APIレスポンスから token・role・employeeId を state にセット |
| `logout()` | state クリア → ログイン画面へリダイレクト |
| ProtectedRoute | AuthContext の token を確認 → 未認証時は LoginPage へリダイレクト |
| Axiosインターセプター | 全APIリクエストに `Authorization: Bearer <token>` を自動付与（課題③の対応） |

### 4.2 打刻画面 — ボタン状態制御（ClockStatus）

| ClockStatus | 活性ボタン |
|-------------|-----------|
| `BEFORE_WORK`（未出勤） | 出勤のみ活性 |
| `WORKING`（出勤中） | 外出・退勤が活性 / 出勤は非活性 |
| `AWAY`（外出中） | 戻りのみ活性 |
| `FINISHED`（退勤済） | 全ボタン非活性 |

### 4.3 Excel取込処理フロー（ImportService）

1. MultipartFile受付 → Apache POI でExcelパース
2. 1行目ヘッダーを読み込み → 列名マッピングテーブルで日英自動マッピング
3. 2行目以降をデータ行として処理
4. 行単位バリデーション実施（必須項目・日付フォーマット・employeeId存在確認）
5. バリデーション通過行をUPSERT（`work_date + employee_id` で既存判定）
6. `ImportResult`（totalCount / successCount / errorCount / errorDetails[]）を返却
7. `batch_import_history` テーブルに処理履歴を記録

**【列名マッピング（日英自動対応）】**

| 日本語列名 | 英語列名 | マッピング先 |
|-----------|---------|------------|
| 社員番号 | `employee_id` | `employeeId` |
| 勤怠日付 | `work_date` | `workDate` |
| 出勤時刻 | `clock_in_time` | `clockInTime` |
| 退勤時刻 | `clock_out_time` | `clockOutTime` |
| 外出時刻 | `go_out_time` | `goOutTime` |
| 戻り時刻 | `return_time` | `returnTime` |
| 業務メモ | `work_memo` | `workMemo` |

## 5. 技術的課題と対応（詳細）

### 5.1 課題① — Springの理解不足（CORS / FilterChain）

- **課題**：Spring Security（CORS / FilterChain設定）の理解不足によりAPIアクセスが通らない
- **対応**：spring.io公式ドキュメント + Baeldungのサンプルで体系的に検証
- **AI活用**：エラー原因の特定にAIを活用し、公式ドキュメントで裏付け確認

### 5.2 課題② — Excel取込エラー（日英ヘッダー混在・不正日付フォーマット）

- **課題**：日本語/英語ヘッダー混在によるCSVマッピング失敗・不正な日付/時刻フォーマット
- **対応①**：ログ出力強化で行番号・エラー種別を特定
- **対応②**：行単位バリデーション追加・成功/失敗件数をレスポンスで明示報告

### 5.3 課題③ — 画面とAPIの連携不具合

- **課題①**：フロント/バック間でレスポンスフィールド名の認識ずれが多発
- **課題②**：JWTトークンの Authorization ヘッダー付与漏れで401エラーが頻発
- **対応①**：API仕様書ベースに命名・型を統一
- **対応②**：AxiosインターセプターでJWT自動付与を実装

## 6. テスト方針

テスト仕様書・テスト結果は本プロジェクトの成果物として作成済み。
（運用マニュアル・保守対応資料は今回対象外。実務では必須成果物と認識。）

| テスト種別 | 対象 | 方法・備考 |
|-----------|------|-----------|
| 機能テスト | 認証 / 打刻 / 集計 / CSV取込 等 | テスト仕様書に機能別テストケース一覧を作成 |
| APIテスト | 全21エンドポイント | Postman / curl による手動テスト |
| セキュリティテスト | 認証・認可 | 無効Token / ロール不一致 / 期限切れTokenでの動作確認 |
| バグ管理 | 全バグ | テスト結果にバグ記録・修正対応履歴を記録 |
