## 1. プロジェクト概要

### 1.1 背景・目的

本プロジェクトは、新入社員の実践的開発スキル向上を目的としたWebシステム開発トレーニングプロジェクトである。

- 新入社員の実践的開発スキル向上
- Java / Spring Boot / React / MySQL を用いた Web システム開発
- 業務想定：勤怠管理＋集計（収益管理は今回対象外）

**【背景・理由】**
- 勤怠管理は業務理解しやすく機能範囲が明確な題材
- 設計〜実装〜テストの一連工程を体験できる
- Java / Spring Boot / React の実務想定トレーニングとして最適

### 1.2 期間・体制

| 項目 | 内容 |
|------|------|
| 開発期間 | 2026年4月1日 ～ 4月30日 |
| 開発メンバー | 開発者 2名、指導責任者 1名（計3名） |
| 開発体制 | 各自のPCで独立開発 → 設計を含む全情報を共有・協議しながら進める（チーム開発の疑似体験） |
| 開発手法 | 簡易的なウォーターフォール方式（要件が初期に確定 → 段階的管理が明確） |

### 1.3 ゴール

- 勤怠管理システムの完成（機能は一部制限あり）
- 設計〜実装〜テストまで一通り経験する
- チーム開発の基礎を習得

**【ゴール設定の意図】**
- コア機能（勤怠管理・集計・ログイン）の完成度を優先
- 収益・入金管理はスコープ外とし、品質に集中

## 2. システム概要

### 2.1 システム全体像

- Web画面で入力・管理・集計
- Excel による勤怠データ入力
- Spring Boot API で処理
- MySQL でデータ管理

**【技術スタック選定理由】**

| 技術 | 選定理由 |
|------|----------|
| React | コンポーネント再利用性が高くSPAに最適・採用市場での需要も高い |
| Spring Boot | Java企業標準フレームワーク・Security / JPA連携が容易 |
| MySQL | OSS無償・Spring Data JPAとの親和性が高く参考資料も豊富 |

### 2.2 構成図

**【本来のシステム構成】**
- クライアント：Windows（Excel / ブラウザ）
- サーバ：Linux（Spring Boot / Tomcat）
- 接続：TeraTerm（SSH）
- DB：MySQL

> ※今回はトレーニング環境のため各自PCで代替実施

**【データフロー】**

```
Browser → Spring Boot API → MySQL（実務と同等の構成を再現）
```

## 4. 非機能要件

| カテゴリ | 要件 | 詳細 |
|----------|------|------|
| バリデーション | 行単位エラー検出・成功/失敗件数の明示報告 | CSV/Excel取込時、行番号・エラー種別を特定してレスポンスで報告 |
| エラーハンドリング | GlobalExceptionHandler実装・例外処理の統一 | BusinessException階層・スタックトレース非公開 |
| 保守性（SRP） | Serviceレイヤー分離（単一責任原則） | Controller約70行・ビジネスロジックをServiceに集約 |
| パフォーマンス | N+1問題解消 | JOIN FETCHにより21クエリ→1クエリに最適化 |
| 再利用性 | AuthContextによるprops drilling解消 | 複合インデックス適用 / セキュリティ二重保護構成 |
| セキュリティ | JWT二重検証 | FilterChain（1次）+ @PreAuthorize（2次） |

## 7. 技術的課題と対応

| No | 課題 | 対応 | 詳細 |
|----|------|------|------|
| ① | Springの理解不足（CORS / FilterChain） | 公式ドキュメント + サンプル検証 / AI活用 | spring.io + Baeldungのサンプルで体系的に検証。AIでエラー原因を特定し公式ドキュメントで裏付け確認 |
| ② | Excel取込エラー（日英ヘッダー混在・不正日付フォーマット） | ログ出力 + バリデーション強化 | ログ強化で行番号・エラー種別を特定。行単位バリデーション追加・成功/失敗件数をレスポンスで報告 |
| ③ | 画面とAPIの連携不具合（フィールド名ずれ・401エラー頻発） | API仕様統一 + テスト強化 | API仕様書ベースに命名・型を統一。AxiosインターセプターでJWT自動付与を実装 |
